POST /api/ingest Auth requise

Cœur du pipeline d'ingestion. Reçoit un tableau de prospects bruts (collectés par les scrapers ou envoyés par n8n), les normalise, déduplique, calcule un score règle-métier et les insère en base. Seul point d'entrée de données dans le système.

1
Réception du batchLe body contient prospects: [...] — liste de dicts avec les champs du prospect.
2
Normalisation → HiringProspectpipeline/ingest.py → ingest_raw_dict() convertit le dict en objet typé.
3
Upsert entrepriseINSERT … ON CONFLICT(linkedin_url) DO UPDATE — la compagnie est créée ou mise à jour.
4
Déduplication 30 joursSi une opportunité identique (même entreprise + même techno + même type) existe dans les 30 derniers jours, elle est ignorée.
5
Score règle-métierscorer_rules.py → compute_rule_score() · pays×0.30 + techno×0.25 + secteur×0.25 + boost×0.20 → score 0–100.
6
INSERT opportunityInsertion en table opportunities avec status='new' et le score calculé.

POST /api/classify NLP Auth requise

Classifie un texte libre avec le modèle ML embarqué. Produit 3 labels (secteur, techno, priorité) avec leurs scores de confiance, puis fusionne le score règle-métier avec le score NLP. Si opportunity_id est fourni, les labels sont mis à jour directement en base.

1
Vérification modèleVérifie que app.state.classifier est chargé. Sinon → HTTP 503 (lancer python -m nlp.trainer).
2
Embedding du texteclassifier.py → _embed() · vecteur 384-dim via paraphrase-multilingual-MiniLM-L12-v2 ou TF-IDF 10000-dim en fallback.
3
3 classificateurs parallèlesLogisticRegression (ou LinearSVC en TF-IDF) prédit secteur (12 classes), techno (10 classes) et priorité (3 niveaux).
4
Fusion NLP scorescorer.py → compute_nlp_score() · final = rule_score×0.60 + nlp_score×0.40.
5
Mise à jour DB (si opportunity_id)UPDATE opportunities SET sector_label, tech_label, priority_label, nlp_confidence, nlp_score.

POST /api/generate/message LLM Auth requise

Génère un message LinkedIn personnalisé (et optionnellement un email) pour une opportunité qualifiée. Groq llama-3.1-8b-instant avec Ollama en fallback local. Le lead généré est immédiatement persisté.

1
Chargement opportunitéget_opportunity_by_id() récupère l'opportunité avec les données de l'entreprise via JOIN.
2
Génération message LinkedInllm_client.py → generate_linkedin_message() · prompt structuré en français, max 5 lignes, 0 formule générique.
3
Génération email (optionnel)Si generate_email: true → generate_email() émet un email de prospection complet.
4
Fallback OllamaSi Groq est indisponible (429, timeout, erreur réseau), bascule automatique vers Ollama local (mistral).
5
INSERT leadcreate_lead() persiste le message, l'email et lie le lead à l'opportunité + au contact.

Formule de Score

Score déterministe règle-métier (Phase 1) puis fusion NLP (Phase 2).
